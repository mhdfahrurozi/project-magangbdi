SIMONEV KINERJA — Technical Specification & Architecture Blueprint
Aplikasi: Sistem Monitoring dan Evaluasi Kinerja

Instansi: Balai Diklat Industri (BDI) Medan

Versi Dokumen: 1.0 — 7 Juni 2026

Tujuan: Acuan rekonstruksi total menggunakan AI Builder

1. TECH STACK & ENVIRONMENT
Platform Builder: ZiteJS (React/TypeScript framework, arsitektur mirip Next.js)

Runtime Backend: Cloudflare Workers (serverless edge functions)

Database: Zite Database (PostgreSQL-backed, SDK ORM)

Frontend:

React (SPA, client-side rendering)

TypeScript (strict mode)

react-router-dom — routing berbasis SPA

Tailwind CSS v3 — utility-first styling dengan CSS variable tokens (HSL)

shadcn/ui — komponen UI (Button, Dialog, Select, Tabs, Card, Checkbox, ScrollArea, Skeleton, Sonner toast, dsb.)

lucide-react — icon pack (LogIn, LogOut, Printer, Eye, Pencil, RefreshCw, dsb.)

date-fns — formatting tanggal

zite-auth-sdk — useAuth() hook, loginWithRedirect(), logout()

zite-endpoints-sdk — typed RPC calls ke backend (pengganti fetch)

Backend (src/api/*.ts):

zod — validasi input/output schema

zite-integrations-backend-sdk — ORM classes per tabel, createEndpoint(), ZiteError

ZitePdf (built-in) — HTML → PDF rendering via headless Chromium

Konfigurasi Penting:

Access Mode: external (publik bisa akses tanpa login, login opsional)

User Sync: enabled — user otomatis tersinkron ke tabel Users saat registrasi

Bahasa HTML: lang="id", translate="no", class notranslate — mencegah browser auto-translate

Font: Inter, fallback -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif

4. CORE FEATURES & BUSINESS LOGIC
4.1 Halaman Publik (/)
Dashboard read-only menampilkan tabel capaian kinerja tahun berjalan + perbandingan 3 tahun terakhir

Filter Tahun di navbar (dropdown)

Visitor Counter:

Frontend: membuat visitorId unik via crypto.randomUUID(), disimpan di localStorage (simonev_visitor_id)

Backend: trackPageView endpoint — cek visitorId terhadap daftar di field visitorIDs (comma-separated). Jika sudah ada → tidak increment, hanya return count. Jika baru → increment viewCount, append visitorId (max 5000 entries)

Efek: refresh/navigasi panel tidak menambah counter; hanya browser/device baru yang dihitung

Tombol Login/Logout responsif di navbar

Generate PDF: tombol window.print() untuk versi cetak halaman publik dengan header cetak khusus

4.2 Modul Perencanaan Kinerja (Admin)
CRUD Tujuan & Sasaran: tipe Tujuan (TJ) atau Sasaran (SK), difilter per tahun

CRUD Indikator Kinerja: linked ke TujuanSasaran, support tipe IKU/IKK, satuan, target, multi-tim, file format

Rencana Aksi per Triwulan: per indikator, input targetFisik (%) dan rencanaAksi (teks) untuk Triwulan I–IV → disimpan di IndicatorPlans

4.3 Modul Pengukuran Kinerja (Admin)
Filter Tahun global

Tab Capaian Kinerja: Admin input realizationValue per indikator per triwulan → disimpan di IndicatorRealizations. Juga support input1 & input2 untuk kalkulasi khusus

Tab Realisasi Anggaran:

Input: paguAwal (Rp) dan realisasiAnggaran (Rp) per indikator per tahun

Rumus Otomatis: capaianAnggaran = (realisasiAnggaran / paguAwal) × 100 — dihitung server-side di endpoint saveIndikatorAnggaran

Disimpan di tabel IndikatorAnggaran

Tab Efisiensi & Efektivitas (Read-Only):

Sinkronisasi otomatis: membaca data dari IndikatorAnggaran (Tab Realisasi Anggaran) dan menghitung:

Capaian Output (%) = (Realisasi / Target) × 100

Capaian Anggaran (%) — diambil dari tabel IndikatorAnggaran

Efisiensi (%) = (Capaian Output / Capaian Anggaran) × 100

Label: ≥110% Sangat Efisien, ≥90% Efisien, ≥80% Cukup Efisien, <80% Tidak Efisien

4.4 Modul Pelaporan Kinerja (Admin & Kepala Balai)
Tab Validasi:

Daftar submissions dari Pelaksana, filter by status

Detail modal: lihat semua field, progress bar, link dokumen

Aksi: Setujui atau Tolak (wajib isi catatan penolakan + opsional link dokumen rapat)

Membuat record di tabel Validations + update Submissions.status

Tab Cetak PDF — Logika 2 Jalur:

3 Bagian Laporan (checklist):
Capaian Tahunan — tabel rekapitulasi per TujuanSasaran

Matriks Triwulan — matriks detail per triwulan (banyak kolom)

Efisiensi & Efektivitas — tabel analisis efisiensi

Jalur 1 (Portrait A4): Jika Capaian Tahunan dan/atau Efisiensi dicentang → digabung ke SATU file PDF portrait. Jika keduanya → page-break di antara

Jalur 2 (Landscape A4): Jika Matriks Triwulan dicentang → file PDF TERPISAH landscape

Info banner biru: muncul saat Matriks + bagian lain dicentang bersamaan: "Matriks Triwulan akan diunduh secara terpisah dalam format Landscape..."

Implementasi: window.open() per orientasi → HTML dokumen mandiri dengan inline CSS + @page { size: A4 portrait/landscape }

Layout Landscape Optimasi:

table-layout: fixed; width: 100%

Font: 8pt, padding: 4px 3px

Kolom: No (3%), Sasaran (13%), Indikator (14%), sisanya proporsional per triwulan

word-wrap: break-word; overflow-wrap: break-word; vertical-align: top

Page-Break Rules: page-break-inside: avoid; break-inside: avoid pada <tr> dan blok tanda tangan

Tanda Tangan Dinamis:

Endpoint getKepalaBalaiName → query Users table cari user dengan role === 'Kepala Balai', return nama

Ditampilkan di footer laporan: "Kepala Balai Diklat Industri Medan" + nama dinamis + garis NIP

4.5 Modul Monitoring Kinerja (Admin & Kepala Balai)
Tabel overview per indikator: target vs realisasi per triwulan, capaian %, color-coded bars

Modal detail per indikator: timeline submissions, plans, dan realizations

4.6 Panel Pelaksana
Input Monitoring: Pelaksana memilih indikator (difilter by team), mengisi progress, kendala, tindak lanjut, uraian rencana aksi, uraian realisasi, dan upload link dokumen → submit ke Submissions dengan status Pending

Hasil Laporan Saya: Daftar submissions milik user. Bisa lihat detail, cetak individu (modal PrintLaporan), dan edit + kirim ulang jika status Rejected

5. UI/UX LAYOUT STANDARDS
Tema Warna
Primary: hsl(217 91% 38%) — Royal Blue instansi ≈ #1054A6

Secondary: hsl(214 32% 91%) — Light blue-gray

Accent: hsl(214 100% 93%) — Very light blue

Background: Pure white hsl(0 0% 100%)

Foreground: Dark navy hsl(222 47% 11%)

Muted: hsl(210 40% 96%) dengan foreground hsl(215 16% 47%)

Destructive: hsl(0 84% 60%) — Red untuk error/tolak

Border: hsl(214 32% 91%)

Border Radius: 0.5rem (8px) base

Semua warna menggunakan CSS variables — WAJIB pakai Tailwind token classes (bg-primary, text-foreground, border-border), DILARANG hardcode hex/rgb

Chart Colors (5 variabel)
chart-1: Royal Blue (primary)

chart-2: Green hsl(142 71% 45%)

chart-3: Orange hsl(38 92% 50%)

chart-4: Red hsl(0 84% 60%)

chart-5: Purple hsl(263 70% 50%)

Struktur Layout
Halaman Publik: Full-width, navbar bg-primary di atas, konten centered max-w-5xl

Panel Internal (Admin/Kepala Balai/Pelaksana): Layout 2 kolom fixed:

Sidebar kiri: w-64, bg-primary, menu navigasi vertikal, logo "SIMONEV KINERJA / BDI Medan", nama user, tombol "Halaman Publik" + "Keluar"

Main content kanan: flex-1 overflow-auto, scrollable, padding p-6

Responsive: Navbar publik menyesuaikan mobile (teks disingkat, padding dikecilkan). Panel internal menggunakan fixed sidebar (desktop-oriented).

Komponen UI Standar
Tabel data: bg-card rounded-xl border shadow-sm, header bg-muted/30, zebra-striping bg-muted/20

Badge status: Hijau (Approved), Kuning (Pending), Merah (Rejected) — masing-masing bg-{color}-100 text-{color}-700

Card metriks: bg-card rounded-xl border shadow-sm p-5

Dialog/Modal: shadcn Dialog dengan max-h-[90vh] scroll, header border-b, footer border-t

Toast: Sonner dengan richColors

Loading: Skeleton loaders sesuai konten shape

Aturan Khusus
Browser Auto-Translate Disabled: <html lang="id" translate="no" class="notranslate"> — diterapkan via useEffect di App.tsx root

Print CSS: @media print rule di index.css menyembunyikan .print\\:hidden, menampilkan .hidden.print\\:block, hide nav
